The following are different deployement on a CI/CD DevOp pipeline

  • DEV: Development

    This is the start where things are being developed

  • SIT: System Integration Testing

    (This is an evivronment in the CI/CD process where developers test different components and modules to work as expected. It is like a place where you test when introducing a new feature and don't want it to intefer with previous features.

  • UAT: User Acceptance Testing

    (This is where a someone who test code-QA, test your code in the organization. iT FOCUS on the user experince and the need of the business how user interact with it)

  • PREPROD/DEMO: Pre-production

    (This is where after everything has been tested working, it's is then push to production. It serves as a the final check before production.)

  • PROD: Production
